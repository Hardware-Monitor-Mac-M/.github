## 💡 Overview

<a href="#hardwareMonitor">Hardware Monitor</a> is a comprehensive hardware sensor monitoring application for Mac — reading and displaying the temperature, fan speed, voltage, current, and power data from the Mac's internal sensors to provide complete visibility into what the hardware is actually doing during normal operation and demanding workloads.

The <a href="#hardwareMonitor">temperature sensor coverage</a> reads CPU core temperatures, GPU temperature, memory temperature, SSD and hard drive temperature, logic board temperature sensors, battery temperature, and any other temperature points the Mac's hardware exposes — providing a complete thermal picture of the machine's operating conditions. Understanding which components are running hot, whether temperatures spike under specific workloads, and how the thermal behavior changes with ambient conditions all depend on the detailed sensor data that Hardware Monitor surfaces. The <a href="#hardwareMonitor">fan speed monitoring</a> reads RPM from all fans in the Mac — the fan or fans that control thermal management under different load conditions. Watching fan RPM alongside temperature readings shows how the Mac's thermal management system is responding to heat load, confirming that cooling is working correctly or identifying situations where fans aren't running as expected given the thermal readings.

The <a href="#hardwareMonitor">battery health monitoring</a> reads the battery's current charge capacity versus design capacity, cycle count, charge state, current draw, and temperature — providing the complete battery health picture that determines whether the battery is performing normally, degrading faster than expected, or approaching replacement. Battery cycle count and capacity percentage are the two key metrics for assessing whether battery replacement is warranted. The <a href="#hardwareMonitor">power consumption readings</a> measure actual power draw from the system — total system power, per-component power where available, and the power efficiency that Apple Silicon Macs expose through their detailed power measurement infrastructure.</p>

<a href="#hardwareMonitor">Configurable threshold alerts</a> notify when any sensor reading exceeds a user-defined threshold — temperature alerts when a component runs hotter than expected, fan alerts when RPM drops below expected range, battery alerts when capacity drops below a threshold. <a href="#hardwareMonitor">Historical logging</a> records sensor data over time — reviewing temperature trends across a workday, tracking fan speed history during a video rendering session, or comparing battery capacity readings over weeks to measure degradation rate.

<a href="#hardwareMonitor">Menu bar display</a> shows selected sensor readings in the Mac menu bar — monitoring CPU temperature, fan speed, or battery percentage from the menu bar without opening the full Hardware Monitor window. <a href="#hardwareMonitor">Multiple display views</a> organize the sensor data in different presentations — a compact overview, detailed individual sensor views, history graphs, and the menu bar display for different monitoring contexts.</p>

Hardware Monitor's thermal monitoring serves a different purpose depending on the Mac model. On MacBook models, thermal throttling is a real and common behavior — sustained CPU and GPU workloads can push temperatures high enough that the Mac reduces clock speeds to protect components, degrading performance in ways that aren't otherwise visible to the user. Hardware Monitor makes this behavior visible and quantifiable, distinguishing thermal limitation from software performance problems.
